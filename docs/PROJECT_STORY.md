## 为什么做这个项目

这个项目的起点并不是“为了做一个通用的 AI 绘图平台”，而是一个非常具体、真实的需求：

- 2026 年元旦，和老同学聚会时聊到科研写作工作流；
- 他在中科院从事农业课题申报相关科研工作，每年有稳定且高频的论文插图需求；
- 当时他多次催我做一个“能持续可用”的自动出图工具；
- 聚会后我就开始动手实现。

中间开发被 `localminidrama`（短剧生成方向）打断过几次，但项目没有放弃。  
现在选择先开源，是为了让协作对象和社区都能直接看到进度，而不是靠口头同步。

## 项目定位

`智绘论文图 / PaperSketch JS` 的定位是：

- 作为 `PaperBanana` 和 `PaperVizAgent` 的 JavaScript 改进增强版本（JS Edition+）；
- 提供更贴近 Web 交付的工程形态（Vue + Node.js）；
- 强调“可运行、可追踪、可持续迭代”，而不是一次性 Demo。

## 开发里程碑（简版）

1. **基础重写阶段**
   - 从 Python 版本抽取 Agent 流水线思想，完成 Node.js 服务端编排；
   - 完成 Vue 前端输入、候选展示、实时进度。
2. **工程化阶段**
   - 引入 SQLite 历史记录与本地图片落盘；
   - 支持中断恢复、断线重连后的状态同步。
3. **模型扩展阶段**
   - 支持 Gemini、fal.ai、Doubao 多模型切换；
   - 前端模型可用性可视化（未配置密钥则提示暂不可用）。
4. **可追踪阶段**
   - 增加每个候选的步骤日志（Planner/Stylist/Critic 等中间过程）；
   - 历史详情可回看中间阶段演化。
5. **Plot 路线调整**
   - 从前端回传渲染切换为后端 Plotly headless 渲染，减少交互链路与脆弱点。

## 为什么先开源

- 让使用者（尤其是最初需求方）可随时看到真实进度；
- 让问题、讨论、迭代决策可追溯；
- 降低“单人维护 + 口头同步”的沟通成本；
- 为后续社区共建预留空间（Issue / PR / 文档反馈）。

## 当前状态

当前已具备：

- 多 Agent 生成流程；
- 多候选并发；
- 历史记录 + 本地持久化；
- 任务恢复与步骤日志；
- 多模型切换与可用性提示；
- Plot 后端渲染链路。
